# Codex Implementation Rules (Draft)

このファイルは、Codex がこのリポジトリで作業する際の実装ルールたたき台です。  
運用しながら更新してください。

## 1. 適用範囲

- 適用対象: 当レポジトリ配下の作業全体

## 2. 基本方針

- コメント、ドキュメント、コミットメッセージは日本語で記載する
- 変更は小さく、意図が分かる単位で行う
- 既存ファイルの文字コード・命名規則を尊重する
- 不要な自動生成物・一時ファイルはコミットしない

## 3. アクセス境界ルール（ユーザー指定）

- このリポジトリ外のファイルやディレクトリを参照する場合は、事前にユーザー承認を得る
- 参照の必要性と対象パスを明示してから実施する

## 4. Python 実行ルール（ユーザー指定）

- Python を使う処理は、必ず仮想環境 (`venv`) で実行する
- 標準コマンド例:
  - Windows: `scripts\\wp2mdpy\\.venv\\Scripts\\python.exe ...`
  - Linux/WSL: `scripts/wp2mdpy/.venv/bin/python ...`
- `python` / `pip` をグローバル環境で直接実行しない

## 5. 変更前チェック

- 既存ルール (`AGENTS.md`, `README.md`) と矛盾しないか確認する
- 影響範囲が大きい変更は、先に方針を短く共有する

## 6. 変更後チェック

- 可能な範囲で最小限の動作確認を行う
- 実行したコマンドと結果を簡潔に共有する
- 未確認事項があれば明示する

## 7. 今後の追加候補

- コーディング規約（命名・フォーマット・コメント方針）
- テスト方針（ユニット/統合/スモークの最小ライン）
- PR ルール（コミットメッセージ、レビュー観点）
- デプロイ関連の安全ルール
