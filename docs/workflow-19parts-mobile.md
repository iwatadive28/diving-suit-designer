# 19部位優先・スマホ向け再設計 作業手順

## 1. 目的とスコープ
この手順書は、19マスク運用を前提に、以下の修正を安全に実施するための作業順を定義する。

- 19部位の再採番（上から、同高さは前→後 / 左→右）
- 画面レイアウト変更
- プリセット動作修正
- テーマ内ランダムおすすめ機能の追加

## 2. 19部位再採番ルール
- ルール:
  - 身体上部から下部へ並べる
  - 同じ高さ帯は前面を先、次に背面
  - 同一面・同一高さは左を先、右を後
- 実装方針:
  - マスク画像ファイルは変更しない
  - `partSpec.ts` の `id` と表示名を更新する
  - マスク参照は `originalRef` で固定して位置ズレを防ぐ

## 3. 画面レイアウト仕様
### 3.1 PC幅
- 上段:
  - 左: デザインプレビュー
  - 右: 選択部位
- 下段（全幅、上から順）:
  - パネルカラー
  - ステッチカラー
  - プリセット / ランダム
  - 出力 / 共有

### 3.2 スマホ幅
- 1カラム縦積み:
  - プレビュー
  - 選択部位
  - パネルカラー
  - ステッチカラー
  - プリセット / ランダム
  - 出力 / 共有

## 4. プリセットデータ仕様
- `public/config/presets.json`
- `parts` のキーは必ず `1..19`
- 存在しない部位キーは読み込み時に除外し、警告ログを出す

## 5. ランダムおすすめ仕様（テーマ内ランダム）
- `public/config/color-themes.json` を追加
- テーマ定義:
  - `id`, `name`, `colors`, `stitchColors`
- 動作:
  1. テーマ選択
  2. `おすすめ生成` 実行
  3. テーマ対象色から部位ごとにランダム選択
  4. ステッチもテーマ内からランダム選択
  5. 生成結果を現在状態へ反映し、URLにも反映

## 6. 実装順序
1. `types.ts` に `ColorTheme` を追加
2. `config.ts` で `color-themes.json` 読み込みとプリセット正規化を実装
3. `partSpec.ts` の19部位を再採番
4. `suitComposer.ts` を `originalRef` マスク参照へ変更
5. `presets.json` を `1..19` キーで再作成
6. `color-themes.json` を作成
7. `App.tsx` を新レイアウト + ランダム機能対応に変更
8. `styles.css` を新レイアウト向けに更新
9. `Readme.md` を仕様更新
10. `npm run build` / `npm run test`

## 7. 受け入れ基準
- 19部位すべてでクリック部位と着色先が一致
- プリセットが適用される
- テーマ内ランダムで配色とステッチが更新される
- 指定レイアウト順を満たす
- 仕様書PNGの部位一覧が再採番順で表示される

## 8. 手動確認手順（スマホ）
1. 部位タップで選択表示が変わる
2. 3部位以上で色変更し、見た目が即時更新される
3. ステッチカラー変更で線色が変わる
4. プリセットを2種類切り替えて反映を確認
5. テーマを変えてランダム生成し、結果が毎回変わる
6. 仕様書PNGを保存し、部位名・色名が一致する
7. URLコピー→別タブで開き、同一状態で復元される
