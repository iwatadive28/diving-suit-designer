# ユーザーペルソナ定義

## 1. 目的
本ドキュメントは、セミドライスーツ配色シミュレーターの設計判断を揃えるために、主要ユーザー像（ペルソナ）と利用文脈を定義する。

## 2. 一次ペルソナ（最重要）
- 役割: 店舗スタッフ（提案・受注担当）
- 経験: ダイビングスーツの提案経験あり
- 利用端末: スマホ
- 利用場所: 店舗、顧客宅
- 1回の利用ゴール:
  - 顧客と配色を決める
  - 発注に必要な仕様情報を確定する
  - 共有URLまたは仕様書画像でそのまま発注へ渡せる状態にする
- 操作時間目安: 最大30分
- 失敗したくない点:
  - 部位ずれ（表示と実際の発注部位が一致しない）

## 3. 二次ペルソナ
- 役割: ビギナーダイバー（発注する顧客）
- 経験: スーツ発注は不慣れ
- 利用端末: スマホ
- 利用場所: 自宅、店舗
- 1回の利用ゴール:
  - 自分の好みの色イメージを具体化する
  - 来店前に候補を作り、スタッフに共有する
- 操作時間目安: 最大30分
- 失敗したくない点:
  - どの部位を変更しているか分からない

## 4. 共通ユースケース
1. 顧客がスマホで配色案を作成する
2. 店舗スタッフが同じURLを開いて内容を確認する
3. 必要に応じて微調整して仕様書画像を生成する
4. 発注先に仕様書画像またはURLを送付する

## 5. UX要件（ペルソナ起点）
- スマホ優先UIであること
- クリック/タップした部位と色反映先が一致すること
- 配色の変更結果が即時に視覚反映されること
- 仕様書画像に部位名・色名・作成日時が明記されること
- 共有URLで同一状態を復元できること

## 6. 品質要件（最重要）
- 最優先品質: 部位ずれゼロ
- 判定基準:
  - マスク画像とクリック判定に同一マッピングを使う
  - 表示色と仕様書記載色が一致する
  - スマホ表示で操作不能箇所がない

## 7. 今後の設計方針
- 29部位の厳密分割より、提供済みマスク（19部位）の正確反映を優先する
- 新しい部位を増やす場合は、先に該当マスクを追加してからUIに公開する
- ステッチ機能は専用マスクが用意でき次第、部位と同じマスク駆動方式へ統一する

## 8. 非対象（このペルソナ定義の範囲外）
- 価格計算ロジック
- 在庫・納期管理
- 顧客管理（CRM）
