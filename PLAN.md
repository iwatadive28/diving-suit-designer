# セミドライスーツ配色シミュレーター 実装計画（v2.1）

## 1. ドキュメント情報
- 文書名: セミドライスーツ配色シミュレーター 改修実装計画
- 対象リポジトリ: `diving-suit-designer`
- 最終更新日: 2026-03-04
- 版数: v2.1
- 対象読者: 企画担当・実装担当

## 2. 目的
現行アプリを「店舗接客で実用可能な操作性・共有性」に引き上げる。  
今回の重点は、性能・共有・比較・初心者導線・復帰性の強化。

## 3. スコープ
### 対象（今回実装）
- 低速端末向け軽量モード
- キャッシュ最適化
- 初期表示2秒以内の監視
- 共有強化（短縮URL、期限付き共有、閲覧専用リンク）
- 比較モード（A案/B案）
- 使い方モーダル先頭に「最短3手順」
- 案件名入力の保持
- アクセシビリティ強化
- 自動保存（下書き復元）
- Undo/Redo の固定表示
- 初回ロードのスケルトン表示と即時反映体感向上

### 非対象（今回見送り）
- バックエンドによる正式認証
- 受注システム連携
- 多言語UI
- 3Dモデル化

## 4. 実装方針（重点8項目）
### 4.1 パフォーマンス
- `lightMode` を導入し、低速端末では以下を自動適用
  - プレビュー解像度を1段階低下
  - ハイライト更新頻度を間引き
  - 高コスト描画（必要時のみ）
- 画像・設定JSONはキャッシュ戦略を明確化
  - ハッシュ付き静的資産: 長期キャッシュ
  - 設定JSON: 再検証付き取得
- 初期表示監視
  - `app_boot_ms` を計測し、2秒超過時は `console.warn` を出力
  - 将来は計測値を外部送信できる構造にする

### 4.2 共有強化
- 状態共有は2系統
  1. 既存の長URL（完全互換）
  2. 短縮共有ID（新規）
- 短縮URL仕様
  - `?sid=<id>` 形式
  - ローカルDB（将来サーバー移行可能）で `id -> state` を解決
- 期限付き共有
  - `expiresAt` を保持し、期限切れ時は復元不可メッセージ
- 閲覧専用リンク
  - `mode=view` 付与時、編集UIを無効化

### 4.3 比較モード（A/B）
- `stateA` / `stateB` を保持し、ワンタップ切替
- 差分強調
  - 部位ごとに A/B の色差分を判定し、差分部位にバッジ表示
- 出力連携
  - 仕様書PNGに比較情報（A/B差分件数）を表示可能にする

### 4.4 初心者導線
- 使い方モーダル先頭に「最短3手順」を固定表示
  1. 部位をタップ
  2. 色を選ぶ
  3. 保存/共有
- 詳細説明は折りたたみ可能にし、最初の認知負荷を下げる

### 4.5 入力負荷削減
- 案件名は `localStorage` に自動保存
- 次回起動時に自動復元
- 空でも保存可能（現行仕様維持）

### 4.6 アクセシビリティ
- ボタンの最小タップ領域を 44px 以上に統一
- コントラスト比を WCAG AA 相当へ調整
- 片手操作向けに主要操作を上部/下部の届きやすい位置へ再配置
- フォーカス可視化とキーボード操作対応を維持

### 4.7 復帰性
- 自動保存（下書き復元）
  - 変更のたびに `draft_state` を保存
  - 起動時に下書き有無を判定し復元
- Undo/Redo は常時視認可能位置に固定

### 4.8 体感高速化
- 初回ロード時にスケルトンUI表示
- 設定・画像ロード完了後に即置換
- 色変更時は先にUI反映し、重い処理は非同期追従

## 5. データ仕様追加
- `AppDraft`
  - `state: SuitState`
  - `savedAt: string`
- `ShareEntry`
  - `id: string`
  - `state: SuitState`
  - `readonly: boolean`
  - `expiresAt?: string`
- `CompareState`
  - `active: "A" | "B"`
  - `A: SuitState`
  - `B: SuitState`

## 6. 画面仕様追加
- ヘッダー
  - Undo / Redo 固定表示
  - 比較モード切替（A/B）
- 保存・共有
  - 通常URLコピー
  - 短縮URL作成
  - 閲覧専用リンク作成
  - 期限付き共有作成
- 起動時
  - スケルトン表示
  - 下書き復元導線

## 7. 受け入れ基準
1. 初期表示計測値が通常端末で2秒以内（ローカル基準）
2. A/B切替が1タップで可能
3. 差分部位が視認できる
4. 案件名が再訪時に復元される
5. 下書き復元が動作する
6. 短縮URL、期限付き共有、閲覧専用リンクが動作する
7. 使い方モーダル先頭に「最短3手順」が表示される
8. 主要操作ボタンがタップしやすく、コントラスト改善が確認できる

## 8. テストケース
### 単体
- 共有IDの生成/解決
- 期限切れ判定
- A/B差分検出
- draft保存/復元
- 案件名保存/復元

### 結合
- 色変更 -> 自動保存 -> 再起動復元
- A案編集 -> B案編集 -> 切替と差分表示
- 閲覧専用リンクで編集不可

### UI
- スマホ片手操作で主要操作へ到達できる
- スケルトンから本描画への遷移が自然
- 低速モードで操作遅延が体感改善する

## 9. 実施順序
1. 下書き復元・案件名復元・Undo/Redo固定表示
2. スケルトン表示・軽量モード・計測
3. 比較モード（A/B）
4. 共有強化（短縮/期限/閲覧専用）
5. アクセシビリティ最終調整
