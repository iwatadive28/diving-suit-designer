# PLAN.md 作成計画（現行実装ベース仕様書）

## 1. ドキュメント情報
- 文書名: セミドライスーツ カラーシミュレーター 仕様・実装計画書
- 対象リポジトリ: `diving-suit-designer`
- 最終更新日: 2026-03-01
- 版数: v1.0
- 対象読者: 企画担当・実装担当

## 2. 目的
セミドライスーツの配色シミュレーションを Web で行い、PNG 出力する MVP を定義する。

## 3. スコープ
### 対象
- パーツ色変更
- プリセット適用
- 画像ダウンロード

### 非対象
- 認証
- サーバー保存
- SNS 共有
- サイズ選択
- 素材選択
- 多言語

## 4. 現状実装のシステム構成
- 単一 HTML 構成（`index.html`）
- 依存ライブラリ（CDN）
  - TailwindCSS
  - html2canvas@1.4.1
- 画像アセットのレイヤー合成で表示

## 5. 画面仕様
- タイトル表示
- キャンバス領域（最大 500px、正方形、背景 + 9 レイヤー）
- プリセットボタン 2 種
- ファイル名入力欄とダウンロードボタン
- パーツ別ドロップダウン（JavaScript で動的生成）

## 6. 機能仕様
### カラー選択機能
- 各パーツの選択色に応じて `images/{partId}/{color}.png` に切り替える。

### プリセット適用機能
- 2 プリセットを一括反映する。

### 画像ダウンロード機能
- `.canvas-container` を html2canvas で PNG 化する。
- ファイル名入力あり: `{入力値}.png`
- 入力なし: `スーツ_YYYYMMDD.png`

## 7. データ仕様
### ColorOption
- `name: string`
- `color: string`（HEX）

### PartInfo
- `id: string`
- `label: string`
- `options: ColorOption[]`

### PresetMap
- `{ [presetName: string]: { [partId: string]: colorName } }`

### 現行定義
- 全 12 色（`fullColorOptions`）
- 膝パッド 2 色（`kneespadColorOptions`）
- パーツ 9 部位（`parts1`, `parts2`, `fastener`, `waist`, `crotch`, `thigh`, `kneespad`, `knees`, `calf`）
- プリセット 2 種類（`ブルー×ホワイト`, `ブラック×レッド`）

## 8. 公開インターフェース
- グローバル関数
  - `applyPreset(presetName: string): void`
  - `downloadSuit(): void`
- 画像パス契約
  - `images/{partId}/{colorName}.png` が存在しない場合は表示欠損となる。

## 9. 非機能要件
- 対応ブラウザ: 最新 Chrome / Edge 推奨
- 配信形態: 静的ホスティング（Vercel / Netlify / GitHub Pages）
- CI 品質ゲート
  - `develop` ブランチ push 時に `tidy -errors -q index.html` を実行

## 10. 既知課題
- 初期ドロップダウン表示が `black` 固定で、初期レイヤー画像（`none` 含む）と不一致
- ドロップダウン外クリックで閉じない
- CDN 依存によりオフライン利用不可
- 画像欠損時のエラーハンドリング未定義
- 一部ラベル文言（例: `また`）の業務用語としての妥当性未確認

## 11. 受け入れ基準
- 全 9 部位で色変更が即時反映される
- 2 プリセットが期待色で反映される
- PNG 出力が表示中デザインと一致する
- ファイル名入力あり / なし双方で命名仕様を満たす
- モバイル幅 / デスクトップで主要レイアウト崩れがない
- CI の HTML lint がエラーなしで通る

## テストケースとシナリオ
### 1. 単体観点
- パーツ 1 つ変更時に対象レイヤーのみ切り替わる
- 存在しないプリセット名指定時に状態不変
- 空文字ファイル名時に日付フォーマット命名される

### 2. 結合観点
- プリセット適用後に個別色変更しても矛盾なく上書きできる
- 連続で色変更した最終状態がダウンロード画像に反映される

### 3. UI 観点
- ドロップダウンの開閉と選択が可能
- スマホ幅でも操作可能（ボタン・入力欄・選択欄が視認可能）
